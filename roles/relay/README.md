# 角色：relay（公网中继）

跑在**任意公网服务器**上，作用是把内网控制面的端口反向暴露到公网。**极轻量**——只用系统自带的 sshd，不装 Docker、不跑任何重服务，适合脆弱的低配 ECS。

## 部署

```bash
# 在公网服务器上（root）
sudo bash setup-relay.sh
```

脚本做两件事：
1. 建一个隧道专用账号 `relay`（比用 root 建隧道安全）
2. 开启 sshd 的 `GatewayPorts`，让反向隧道能绑定到公网网卡

## 之后

- 把**控制面机器**的隧道公钥加到本机 `/home/relay/.ssh/authorized_keys`
- 在**安全组/防火墙**放行对外端口（默认 `8443`）
- 控制面那台机器起 `devsys-tunnel` 服务后，公网 `:8443` 就会转发到内网 Coder

## 换公网服务器

这台机器**无状态**。换一台公网服务器，只需在新机器上重跑 `setup-relay.sh`，再把控制面 `.env` 里的 `RELAY_HOST` 改成新地址、重启隧道即可。不涉及任何数据迁移。
