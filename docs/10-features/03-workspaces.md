# 持久工作区

**工作区**基于 `tmux`,用来跑**不能中断**的任务 —— 训练、长时间的数据处理等。

## 新建

在工作区页点右上角 **+**,选服务器、起个名字、新建。随后在终端里照常跑命令:

```bash
python train.py          # 训练
tail -f run.log          # 看日志
```

## 关掉网页会怎样

关闭网页只是**断开**,进程继续在服务器上跑。回到工作区页点**打开**即可继续接入,
终端历史与正在运行的命令原样还在。

## 与 tmux 的关系

- 工作区列表与你在机器上 `tmux ls` 看到的是**同一批**会话。
- 打开会话是**接管**模式:若该会话正挂在别处,会被切到当前浏览器。

## 何时不该用普通 SSH

在普通 [Web SSH](01-ssh) 里跑长任务,关页就断。凡是要跑很久的,一律用工作区。
