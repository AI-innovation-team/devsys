# 高速上传文件 / 目录

在**工作区**页点右上角的 **↑ 上传** 按钮,展开上传面板,即可把本地的文件或**整个目录**直接送到内网服务器上——无需 `scp`、无需命令行。

## 怎么用

1. 选**目标服务器**(下拉,只列你已配好凭据的机器)。
2. 填**目标目录**:
   - `~` 或留空 → 你在该机的家目录
   - `~/proj` 或 `proj` → 家目录下的 `proj/`(不存在会自动创建)
   - `/data/xxx` → 绝对路径
3. 把文件或文件夹**拖进虚线框**,或点「选文件 / 选文件夹」。

上传会即时显示进度、速率与预计剩余时间。**断了会自动重试续传**,不用从头再来。文件落到服务器后,你在终端 / 工作区里 `ls` 就能看到。

## 它快在哪、慢在哪(实话)

- **压缩**:上传前在浏览器里 gzip。对代码、配置、文本这类内容,体积常常缩到几分之一,等于把带宽乘了好几倍。
- **打包**:整个目录先打成**单条 tar 流**再传,消灭了成百上千个小文件逐个握手的开销。
- **续传**:分块上传 + 断点续传,专为脆弱的公网入口设计,传大文件也不怕中途掉线。

但有个物理上限得讲清楚:所有流量都要穿过唯一的公网入口(relay),它的**出口带宽**就是天花板。**代码 / 配置**这类可压缩内容,压缩后很快;而**已压缩的大文件**(模型权重、视频、`.zip` 等)压不动,传 GB 级会是分钟到小时级——这不是软件能绕开的,是那条公网链路的物理限制。这种超大文件,若你本机在 Tailscale 内网里,直接 `rsync` 到目标机会更快。

## 限制

- 依赖较新的浏览器(Chrome / Edge / Safari 近两年版本);过老的浏览器会提示不支持。
- 单文件上限约 8 GiB(tar 格式字段所限),常规使用足够。
- 目标目录的写入权限就是你自己的账号权限——你在终端能写的地方,这里就能传。
