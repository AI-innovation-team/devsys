# 使用指南

欢迎使用 **devsys 内网门户**。用你自己的 GitHub 账号登录后，即可从浏览器直达内网服务器 —— Web SSH、VS Code，以及可持久化的工作区。

## 第一步：配置连接凭据

点左下角**头像 → 设置**，在「连接凭据」里为每台服务器填写：

- **用户名 (userid)**：你在该机上的 Linux 账号
- **认证方式**：密码 或 SSH 私钥
- 保存后，凭据经 **Fernet 加密** 存于门户，仅用于以你的身份发起连接

> 门户不持有 root，也不共享凭据 —— 每个人用自己的账号连接。

## 第二步：连接服务器

在**服务器**页，每台机有两个入口：

| 入口 | 说明 | 是否持久 |
| --- | --- | --- |
| SSH | 浏览器终端，临时会话 | 否，关页即断 |
| VS Code | 浏览器版 VS Code | 是，实例常驻 |

## 第三步：工作区（持久会话）

**工作区**基于 `tmux`，用来跑那些不能中断的任务：

```bash
# 在工作区里正常使用即可，例如
python train.py            # 训练
tail -f run.log            # 看日志
```

关掉网页只是**断开**，进程继续在服务器上跑。回到工作区页点**打开**即可重新接入，终端历史与正在运行的命令原样还在。

- 工作区列表与你在机器上 `tmux ls` 看到的是**同一批**
- 打开会话采用**接管**模式：若该会话正挂在别处，会被切到当前浏览器
