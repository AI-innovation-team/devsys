# 使用指南

欢迎使用 **devsys 内网门户**。登录后即可从浏览器直达内网服务器 —— Web SSH、VS Code，以及可持久化的工作区。

## 登录

门户为**申请制**，两种登录方式（需要账号请联系管理员）：

- **GitHub**：用你的 GitHub 账号一键登录。
- **邮箱**：管理员为你开通邮箱账号，初始密码 `modifyme2026`。首次登录后请到 **设置 → 账户** 改成自己的密码。

> 邮箱登录和 GitHub 登录是**两个独立身份**，各自的服务器凭据、工作区互相独立。建议固定用一种。

## 第一步：配置连接凭据

点左下角**头像 → 设置**，在「连接凭据」里为每台服务器填写：

- **用户名 (userid)**：你在该机上的 Linux 账号
- **认证方式**：密码 或 SSH 私钥
- 保存后凭据经 **Fernet 加密** 存于门户，仅用于以你的身份发起连接

> 门户不持有 root、不共享凭据 —— 每个人用自己的账号连接。

## 第二步：连接服务器

在**服务器**页，配好凭据的机器有两个入口：

| 入口 | 说明 | 是否持久 |
| --- | --- | --- |
| SSH | 浏览器终端，临时会话 | 否，关页即断 |
| VS Code | 浏览器版 VS Code | 是，实例常驻 |

没配凭据的机器会显示「设置凭据」按钮，点它去设置。

## 第三步：工作区（持久会话）

**工作区**基于 `tmux`，用来跑不能中断的任务。点右上角 **+** 选服务器、起名、新建：

```bash
python train.py            # 训练
tail -f run.log            # 看日志
```

关掉网页只是**断开**，进程继续在服务器上跑。回到工作区页点**打开**即可继续接入，终端历史与正在运行的命令原样还在。

- 工作区列表与你在机器上 `tmux ls` 看到的是**同一批**
- 打开会话是**接管**模式：若该会话正挂在别处，会被切到当前浏览器

## 设置里还有什么

- **账户**（仅邮箱登录可见）：修改登录密码
- **外观**：浅色 / 深色主题

需要帮助？联系管理员。
